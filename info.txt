Project Title:
Garbage Collection Management System - Frontend Web UI

Project Goal:
Build a modern, responsive frontend-only web application for a private garbage collection company.
This phase is UI-first with mock data and local state only (no backend).

Confirmed Direction (Locked):
- Use the existing template structure as-is.
- Keep current navigation, sidebar behavior, and layout patterns from template.
- No role-based access logic for now.
- Admin, Driver, Accountant, Support, Supervisor are menu groups only.
- Open directly to dashboards (no login flow in this phase).
- Single-tenant only.
- One shared sidebar only (existing sidebar).
- Light mode only (static).

--------------------------------------------------
FOUNDATION RULES
--------------------------------------------------

1) Frontend-only
- Do not build backend endpoints.
- Do not add real authentication.
- Do not add database connections.

2) Data
- Use centralized static mock data files.
- Keep IDs and entity relationships realistic.
- Prepare data shape to be easy to replace with APIs later.

3) UI
- Reuse template styles and component patterns.
- Keep responsive behavior aligned with template standards.
- Add loading, empty, and basic error UI states on data views.

4) Architecture readiness
- Keep a service layer for data access (even if mock only).
- Use consistent response shape in mock services:
  - Success: { code: 1, data, message }
  - Error: { code: 2, message }
